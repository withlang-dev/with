# `with get`

```
with get color                      # first-party (QuixiAI/color)
with get someone/lib                # third-party (github.com/someone/lib)
with get c.box2d/3.0                # C package (via Conan)
```

---

## 1. Package Resolution

```
color                → github.com/QuixiAI/color       no slash = first party
someone/lib          → github.com/someone/lib          slash, no dot = GitHub
gitlab.com/x/lib     → gitlab.com/x/lib                dot = explicit domain
```

**No slash:** First-party package maintained by the With team.
Resolves to `github.com/QuixiAI/<name>`.

**Slash, no dot in first segment:** Third-party GitHub package.
Resolves to `github.com/<owner>/<repo>`.

**Dot in first segment:** Explicit domain. Used as-is.

In code:

```with
use color                    // first party
use someone/lib              // third party
use gitlab.com/x/lib         // non-GitHub
```

Import paths are a syntactic construct, not strings. No quotes.

### 1.1 Version Selection

```
with get color                      # latest tagged version
with get color@v1.2.0               # exact version
with get color@v1                   # latest v1.x.x
with get color@latest               # tip of default branch
with get someone/lib@v0.3           # third-party, pinned
```

Tags must be semver: `v1.0.0`, `v1.2.3`. Untagged repos work
with `@latest` but `with get` warns you.

Minimum version selection. If you say `v1.2.0`, you get `v1.2.0`.
Not "the latest compatible." Predictable. No SAT solver. No
surprises.

`with get -u color` upgrades to latest within current major.

### 1.2 Private Repos

`with get` fetches via `git clone`. If your SSH key or Git
credentials work with `git clone`, they work with `with get`.
No special config.

## 2. C Packages

Everything starting with `c.` routes through Conan.

```
with get c.box2d                # latest
with get c.box2d/3.0            # version 3.0.x
with get c.box2d/3.0.1          # exact
```

The `c.` prefix is stripped, the rest is passed to Conan. In code:

```with
use c_import("box2d/box2d.h", link: "box2d")
use c_import("SDL3/SDL.h", link: "SDL3")
```

`with get` installs headers to `.with/c/include/` and libs to
`.with/c/lib/`. The compiler finds them automatically.

### 2.1 Opinionated Defaults (Tier 1)

For 90% of users, `with get c.box2d` just works:

- **Static linking.** Always. Shared is opt-in.
- **Release profile.** Optimized. No debug symbols in deps.
- **Curated build config.** We maintain known-good configs for
  common C libs. You get the build we tested, not the build
  Conan guessed.
- **No feature flags.** If a library has 50 toggles, we picked
  one configuration.

No flags, no options, no decisions.

### 2.2 Advanced Mode (Tier 2)

For the 10% who need custom builds, use the table form in
`with.toml`:

```toml
[cdeps."c.openssl"]
version = "3.2"
shared = true

[cdeps."c.curl"]
version = "8.5"
options = { with_ssl = "openssl", shared = true }
```

The switch from simple (`"c.box2d" = "3.0.1"`) to table form
is how you opt into advanced mode. Visible in the file.

### 2.3 Local Override

For C libraries not on Conan:

```
with get c.my-lib --path /usr/local/lib/my-lib
```

Expects `include/` and `lib/` subdirectories at that path.

### 2.4 Error Handling

Conan errors are caught and re-emitted as With diagnostics:

```
$ with get c.nonexistent
error: C package 'nonexistent' not found on Conan Center.
  Check: https://conan.io/center
  Or use --path for local installs.

$ with build
error: Failed to build C dep 'sdl3' for linux-x86_64.
  Missing system package: libx11-dev
  Install: sudo apt install libx11-dev
  Then: with build
```

The user never sees a Python traceback. Never needs to know
Conan exists.

## 3. `with.toml`

```toml
[package]
name = "falling-with-style"
version = "0.1.0"

[deps]
color = "v1.2.0"
json = "v0.4.1"
"someone/lib" = "v1.0.0"
"gitlab.com/x/lib" = "v2.0.0"

[cdeps]
"c.box2d" = "3.0.1"
"c.sdl3" = "3.1.2"
"c.sdl3_mixer" = "3.1.0"
```

First-party packages are just a name. Third-party packages
include the owner. Non-GitHub packages include the domain. You
can see at a glance what's official and what isn't.

## 4. `with.lock`

Auto-generated. Committed to version control. Non-negotiable.

```toml
# AUTO-GENERATED by with 0.1.0

[deps]
color = { version = "v1.2.0", source = "github.com/QuixiAI/color", hash = "sha256:abc..." }
json = { version = "v0.4.1", source = "github.com/QuixiAI/json", hash = "sha256:def..." }
"someone/lib" = { version = "v1.0.0", source = "github.com/someone/lib", hash = "sha256:ghi..." }

[cdeps]
"c.box2d" = { version = "3.0.1", conan_ref = "box2d/3.0.1#rev789", package_id = "abc..." }
"c.sdl3" = { version = "3.1.2", conan_ref = "sdl3/3.1.2#revabc", package_id = "def..." }
```

The lockfile records the fully resolved source for every package.
Builds are reproducible regardless of resolution rules.

## 5. `with tidy`

Separate from `with get`. Analyzes source code, prunes unused
deps, adds missing ones:

```
$ with tidy
removed: json v0.4.1 (not imported)
added:   http v1.0.0 (imported in src/client.w)
removed: c.libpng 1.6.40 (not linked)
```

`with get` always takes an argument. `with tidy` never does.

## 7. Starter Packages

Ship these on day one so the ecosystem has a pulse.

**In stdlib (ships with the compiler):**

`args` — command-line argument parser. Comptime-driven: define a
struct with your flags and `args.parse[Config]()` generates the
parser at compile time from the struct fields. No runtime
reflection. Every CLI tool needs this, so it belongs in the box.
Showcases comptime.

**First external packages (`with get` required):**

`color` — terminal color output. Under 200 lines. The first
package every With user will install. API:
`color.red("error: {msg}")`. Showcases string interpolation and
enums. Small enough that someone can read the entire source in
10 minutes, which makes it the de facto "how to write a With
library" example.

`json` — JSON parse/serialize. Comptime-driven:
`json.decode[Config](str)` maps JSON to a struct at compile time.
500–800 lines. Exercises generics, comptime, error handling, and
string processing.

`http` — minimal HTTP client. Wraps `c.curl` behind a clean
With API. This is the package that proves `with get` and
`c_import` work together end-to-end: someone does `with get http`,
it pulls in `c.curl` as a transitive C dep, and they make an
HTTP request in three lines. Showcases the full dependency chain.
